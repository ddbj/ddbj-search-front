# 外部ツール運用メモ

## この文書の役割

この文書では、AIがNotionやLinearなどの外部ツールを使うときの標準運用を定める。

- 長時間の無応答を避ける
- 途中経過を短く共有する
- 中断や停止の可能性があるときは、成功前提で先に進めない

## 背景

- 外部ツールの呼び出しは、応答が返るまで中間状態が見えにくい
- とくに長時間反応がないときは、ユーザーから見ると処理が止まっているように見える
- Codexはユーザーの中断操作を瞬間的・連続的に監視しているわけではない
- 中断は主に `aborted by user` というツール結果や、後続の `<turn_aborted>` イベントを受け取った時点で事後的に認識する

## 標準運用

- 外部ツールを使う前に、何をしに行くのかを短く共有する
- 外部ツールの応答を無制限に待ち続けない
- 反応がない、または止まっている可能性がある場合は、再試行か状態共有に切り替える
- 同じ呼び出しを根拠なく繰り返して、待ちぼうけ状態を延長しない
- 外部操作が中断された可能性がある場合は、成功前提で先に進めず、現在状態を確認してから再開する
- 「処理中です」のまま長時間放置せず、どこで止まっているかを説明する

## 期待値

- 完全なリアルタイム監視はできない
- ただし、「気づいたら 15 分無反応」のような状態は避けることを優先する
- 特にNotionやLinearなどの外部サービス連携では、処理を進めること以上に、止まっているときに止まっていると伝えることを重視する

## 見直し条件

- 外部ツールの応答特性が大きく変わったとき
- 中断検知や進捗共有の仕組みが改善されたとき
- この運用よりも適切な共通ルールが別文書に移ったとき
