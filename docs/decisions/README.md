# 設計判断ログ

## このディレクトリの役割

このディレクトリでは、個別の設計判断や時点つきの実装方針を管理する。

以下のような内容を対象にする。

- ある時点で採用した設計判断
- 代替案と、その案を採らなかった理由
- 将来見直す可能性がある実装方針
- ドメイン仕様に引きずられた例外対応の背景

以下の内容は、原則としてこのディレクトリでは扱わない。

- 恒久的な命名規約
- 日常開発の手順や検証ルール
- 単なるライブラリ一覧

## 運用ルール

- 1つの判断につき1ファイルで残す
- ファイル名は `YYYY-MM-DD-short-title.md` のように日付と短い要約を含める
- 冒頭で「何を決めたか」を短く書く
- 判断理由、影響範囲、今後の見直し余地を残す
- 恒久ルールになった内容は、後で適切な基準文書へ移す

## ひな形

```md
# タイトル

## 決定

- ここで何を決めたか

## 背景

- なぜ判断が必要だったか

## 採用した理由

- 採用案の利点
- 見送った案との差分

## 影響範囲

- どの画面、機能、運用に影響するか

## 補足（任意）

- 実装上の注意点や関連する技術的な制約など

## 見直し条件

- どんな条件になったら再検討するか
```
